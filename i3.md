# LemonCUC i3 重构计划

## 1. 现有架构

```
浏览器                        Docker 容器
┌─────────────┐              ┌─────────────────────────────────────────┐
│ index.html  │              │  Xvfb :0                                │
│  ├ noVNC ───┤──ws:6080──►  │  ├ XFCE4 (startxfce4)                  │
│  ├ WebAudio─┤──ws:5702──►  │  ├ x11vnc :5900                        │
│  └ toolbar  │              │  ├ PulseAudio (null-sink)               │
└─────────────┘              │  ├ lemon-cuc-backend                    │
                              │  │  ├ websockify :6080 → :5900 (VNC)   │
                              │  │  ├ websockify :5702 → :5701 (audio) │
                              │  │  └ tcpulse :5701 (gst→fMP4)         │
                              │  └ Chrome (.desktop launcher)           │
                              └─────────────────────────────────────────┘
```

### 现有问题

| 问题 | 说明 |
|---|---|
| **多端口** | VNC 用 :6080, 音频用 :5702, 还需分别 EXPOSE / 映射 |
| **XFCE4 臃肿** | xfce4 + xfce4-goodies + xfce4-terminal + tumbler + xfconf 一堆, 镜像很大 |
| **无终端** | 虽然装了 xfce4-terminal 但没有独立的 SSH 通道, 只能靠 VNC 内打开终端 |
| **Chrome 启动方式** | 用 .desktop 文件硬编码, 无法通过配置文件声明式管理应用列表 |
| **无通知转发** | 容器内 notification 无法传到浏览器 |
| **无 AI 友好接口** | 没有暴露 xdotool / dmenu / i3-msg 等 API, AI agent 无法直接操控桌面 |
| **鼠标太小** | 默认 X cursor 在高分辨率下难以看到 |

## 2. 目标架构

```
浏览器                           Docker 容器
┌──────────────────────┐        ┌──────────────────────────────────────────────┐
│ index.html           │        │  Xvfb :0 (固定分辨率, 大鼠标)                │
│  ├ tab: VNC + Audio  │        │  ├ i3-wm (定制 config)                       │
│  │  ├ noVNC ─────────┤─►      │  │  ├ i3bar + i3status                       │
│  │  └ WebAudio ──────┤─►      │  │  └ dmenu (启动器)                          │
│  ├ tab: Terminal(s)  │        │  ├ x11vnc :5900                              │
│  │  └ xterm.js ──────┤─►      │  ├ sshd :22                                  │
│  ├ Notifications ────┤◄─      │  ├ dunst → D-Bus notifications               │
│  └ toolbar           │        │  ├ PulseAudio (null-sink)                    │
└──────────────────────┘        │  ├ lemon-cuc-backend (:6080 单端口)           │
         │                      │  │  ├ GET  /           → 静态前端 (dist/)     │
      ws://host:6080            │  │  ├ WS   /ws/vnc     → :5900 (VNC)         │
         │                      │  │  ├ WS   /ws/audio   → tcpulse (fMP4)      │
         ▼                      │  │  ├ WS   /ws/ssh     → SSH client → :22    │
   单端口, 路径分发               │  │  ├ WS   /ws/notify  → D-Bus listener push │
                                │  │  ├ POST /api/xdotool → xdotool wrapper    │
                                │  │  ├ POST /api/dmenu   → dmenu feeder       │
                                │  │  ├ POST /api/i3msg   → i3-msg proxy       │
                                │  │  └ POST /api/scrot   → screenshot         │
                                │  └ Chrome (via properties.json)              │
                                └──────────────────────────────────────────────┘
```

### 3.2 配置文件 (files/)

| 文件 | 说明 |
|---|---|
| `files/i3-config` | i3 窗管配置: 快捷键, 启动项, 窗口规则, i3bar 配置 |
| `files/i3status.conf` | i3status 状态栏: 时间/内存/CPU |
| `files/dunstrc` | dunst 通知样式配置 |
| `files/sshd_config` | sshd 配置: 只监听 127.0.0.1:22, 允许 root |
| `files/properties.json` | 应用声明: Chrome 及其 flags, 以及其他要预装的应用 |
| `files/Xcursor.theme` | 大鼠标 cursor (或 index.theme 指向 DMZ-White/Adwaita 并放大) |
| ~~files/google-chrome.desktop~~ | **删除** |
| ~~files/screensaver.desktop~~ | **删除** (i3 没有 screensaver) |

#### properties.json 示例

```jsonc
{
  "apps": [
    {
      "name": "Google Chrome",
      "exec": "/usr/bin/google-chrome-stable",
      "args": [
        "--no-sandbox", "--no-first-run", "--disable-gpu",
        "--disable-software-rasterizer", "--disable-dev-shm-usage"
      ],
      "autostart": false
    }
  ],
  "cursor": {
    "theme": "Adwaita",
    "size": 48
  },
  "display": {
    "width": 1280,
    "height": 800,
    "depth": 24
  }
}
```

entrypoint.sh 在启动时读 properties.json, 用 jq 解析应用列表, 生成 i3 exec 行或直接启动 autostart 应用。Rust 后端也读它来知道有哪些可启动的应用。

### 3.4 Rust 后端重构 (src/main.rs)

**当前**: 3 个独立 `tokio::spawn` — 2 个 websockify + 1 个 tcpulse, 各自 `TcpListener::bind`。

**目标**: 单 `TcpListener` on `:6080`, 手动做 HTTP/1.1 解析 + WebSocket upgrade, 根据路径分发。

#### 模块结构

```
src/
  main.rs              # 单端口 HTTP server + WS upgrader + 路由
  ws_router.rs         # 路径匹配 → handler 分发
  handlers/
    mod.rs
    vnc.rs             # /ws/vnc   → websockify 到 :5900
    audio.rs           # /ws/audio → 内部 tcpulse (gst 子进程)
    ssh.rs             # /ws/ssh   → SSH client 到 :22 (PTY 模式)
    notify.rs          # /ws/notify → D-Bus notification listener → WS push
  api/
    mod.rs
    xdotool.rs         # POST /api/xdotool { "action": "type", "text": "hello" }
    dmenu.rs           # POST /api/dmenu   { "items": [...], "prompt": "Run:" }
    i3msg.rs           # POST /api/i3msg   { "command": "workspace 2" }
    scrot.rs           # POST /api/scrot   → 返回 PNG screenshot
    properties.rs      # GET  /api/properties → 返回 properties.json
  tcpulse.rs           # GStreamer pipeline 管理 (从 tcpulse/mod.rs 简化)
```

#### 依赖变更 (Cargo.toml)

```diff
  [dependencies]
  tokio = { version = "1", features = ["full"] }
- tokio-tungstenite = "0.26"
  futures-util = "0.3"
```

> 用 axum — 内置 WS extract, tower-http 提供静态文件, aide 生成 OpenAPI 3 文档。
> `tokio-tungstenite` 不再需要, axum 自带 WS 支持。

#### 路由逻辑

```rust
use aide::axum::{ApiRouter, routing::{get, post}};
use aide::openapi::OpenApi;

fn api_router() -> ApiRouter {
    ApiRouter::new()
        // ── WebSocket upgrades (不进 OpenAPI, 用原生 axum 路由) ──
        .route("/ws/vnc",    get(ws_vnc))
        .route("/ws/audio",  get(ws_audio))
        .route("/ws/ssh",    get(ws_ssh))
        .route("/ws/notify", get(ws_notify))
        // ── AI-friendly REST API (aide 自动生成 OpenAPI doc) ──
        .api_route("/api/xdotool",    post(api_xdotool))
        .api_route("/api/dmenu",      post(api_dmenu))
        .api_route("/api/i3msg",      post(api_i3msg))
        .api_route("/api/scrot",      get(api_scrot))
        .api_route("/api/properties", get(api_properties))
        // ── OpenAPI JSON ──
        .route("/api/openapi.json", get(serve_openapi))
        // ── CORS (前端单独部署, 需要跨域) ──
        .layer(CorsLayer::permissive())
}
```

### 3.5 SSH handler 详细设计

路线 B: 后端做 SSH client, 前端只是 xterm.js。

```
浏览器 xterm.js ↔ WS /ws/ssh ↔ [Rust 后端 SSH client] ↔ sshd :22
```

- 后端收到 WS 连接后, spawn `ssh -o StrictHostKeyChecking=no root@127.0.0.1` (或用 `thrussh` / `russh` crate 直接做 SSH, 避免依赖 ssh 二进制)
- 更简单的方案: 直接走 **裸 TCP 透传** 到 `:22`, 前端用 wasm SSH 库 (如 pake 的 `ssh-wasm`)... 但你说 B, 那就后端做。
- 最简: 后端用 `tokio::process::Command` spawn `ssh` 加 `-tt` (force PTY), stdin/stdout pipe 到 WS。  
  但 ssh 连自己需要密码或 key — 用 `ssh-keygen` 在 Dockerfile 里预生成 root authorized_keys 实现免密。
- **多 session**: 每个 WS 连接 = 一个独立的 SSH 进程, 互不影响。前端 tab 管理。

### 3.6 Notify handler 详细设计

```
应用 → D-Bus org.freedesktop.Notifications.Notify()
                    │
              dunst (显示 toast)
                    │
    lemon-cuc-backend (zbus 监听 signal)
                    │
              JSON over WS → 浏览器 Notification API
```

- 后端用 `zbus` 连接 session bus, 监听 `org.freedesktop.Notifications` 接口的 `Notify` method call (做为 monitor) 或 subscribe `ActionInvoked`/`NotificationClosed` signals。
- 更直接: 后端自己**实现** `org.freedesktop.Notifications` 接口(替代 dunst), 这样所有通知自动经过后端。但你说要 dunst, 那就共存 — 后端监听, dunst 显示。
- WS 推送格式:

```json
{
  "type": "notification",
  "id": 42,
  "summary": "Download complete",
  "body": "file.zip saved",
  "app_name": "Chrome",
  "icon": "dialog-information",
  "timeout": 5000
}
```

- 前端收到后调 `new Notification(summary, { body })` 或自绘 toast。

### 3.7 AI 友好 REST API 详细设计

| 端点 | 方法 | 请求体 | 响应 | 说明 |
|---|---|---|---|---|
| `/api/xdotool` | POST | `{ "action": "type", "args": ["--delay", "50", "hello world"] }` | `{ "ok": true }` | 转发 `xdotool type --delay 50 "hello world"` |
| `/api/xdotool` | POST | `{ "action": "key", "args": ["ctrl+c"] }` | `{ "ok": true }` | `xdotool key ctrl+c` |
| `/api/xdotool` | POST | `{ "action": "mousemove", "args": ["500", "300"] }` | `{ "ok": true }` | `xdotool mousemove 500 300` |
| `/api/xdotool` | POST | `{ "action": "click", "args": ["1"] }` | `{ "ok": true }` | `xdotool click 1` |
| `/api/dmenu` | POST | `{ "items": ["firefox", "chrome", "thunar"], "prompt": "Launch:" }` | `{ "selected": "chrome" }` | echo items \| dmenu -p prompt |
| `/api/i3msg` | POST | `{ "command": "workspace 2" }` | `{ "success": true }` | `i3-msg "workspace 2"` |
| `/api/i3msg` | POST | `{ "command": "exec google-chrome" }` | `{ "success": true }` | `i3-msg exec ...` |
| `/api/scrot` | GET | — | `image/png` 二进制 | `scrot -o /tmp/shot.png && cat /tmp/shot.png` |
| `/api/properties` | GET | — | properties.json 内容 | 应用列表、显示配置 |
| `/api/properties` | PUT | 新 JSON | `{ "ok": true }` | 热更新配置 (可选, v2) |

这些接口让 AI agent 可以:
1. 用 `/api/scrot` 截屏 → 视觉理解当前桌面
2. 用 `/api/xdotool` 精确操控鼠标/键盘
3. 用 `/api/i3msg` 管理窗口布局 (切 workspace, focus, move, resize)
4. 用 `/api/dmenu` 交互式选择/启动应用

### 3.10 大鼠标

在 Dockerfile 中:

```dockerfile
RUN apt-get install -y xcursor-themes
```

在 entrypoint.sh 或 i3 config 中:

```bash
# 从 properties.json 读取或用默认值
export XCURSOR_THEME=Adwaita
export XCURSOR_SIZE=48
```

X cursor size 48px 在 1280x800 下很醒目,不会丢失。
